# Format Code

Format the file or code identified by the user.

## Instructions

1. Read the target file.
2. Fix indentation, spacing, and unnecessary blank lines.
3. Follow the standard formatting conventions for the file's programming language.
4. Preserve the existing behavior.
5. Do not rename functions, variables, classes, or files.
6. Do not add features or refactor the implementation.
7. Do not change comments unless formatting requires it.
8. Make only formatting-related changes.

After editing the file, briefly summarize what formatting was changed.
